.gitignore は、Git に「このファイルは記録しなくていいよ」と伝えるための設定ファイルです。

Git は本来、フォルダの中のファイルを全部バージョン管理（履歴を記録）しようとします。でも実際には「記録したくない・してはいけない」ファイルがあります。そういうファイル名やフォルダ名をここに書いておくと、Git が無視してくれます（git status にも出てこなくなります）。

無視したい理由は大きく3つです。①秘密だから、②自動生成されるから、③自分の環境専用だから。

その観点で今の中身を見ていくとこうなります。

行	何のファイル	なぜ無視するか
.env	APIキーなどの秘密情報	①GitHub に上げたら Anthropic の APIキーが世界中に公開されてしまう
__pycache__/	Python が実行時に作る高速化キャッシュ	②自動生成。消してもまた作られる
*.pyc	同上（キャッシュの中身の1ファイル1ファイル）	②同上。* は「なんでもいい」の意味で、main.pyc などすべてに一致
*.db	SQLite のデータベース本体（data/sakumon.db）	②③実際のデータであってプログラムではない。人ごとに中身が違う
*.db-shm / *.db-wal	SQLite が動作中に一時的に作る作業ファイル	②勝手に作られて勝手に消える
venv/ / .venv/	Python の仮想環境（インストールしたライブラリ本体）	③何万ファイルもある上、requirements.txt があれば誰でも再作成できる
.claude/	Claude Code の設定・履歴	③自分の環境専用
書き方のルールもシンプルで、末尾が / ならフォルダ全体、* は「任意の文字列」のワイルドカード、それ以外はそのままのファイル名です。

特に大事なのが1行目の .env です。APIキーが書かれたファイルが GitHub の公開リポジトリに載ると、悪用されて高額請求が来ることが実際によくあります。この1行がそれを防いでいます。

ひとつ注意点として、.gitignore は「まだ Git に記録されていないファイル」にしか効きません。一度 git add して記録済みのファイルは、後から .gitignore に書いても無視されず追跡され続けます。その場合は git rm --cached ファイル名 で追跡を外す必要がありま
